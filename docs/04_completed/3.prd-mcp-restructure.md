# Revise PRD for MCP-First Architecture

⚠️ **Medium** — Documentation-only changes across 4 files, but requires careful consistency across all module references.

## Objective

Restructure the PRD, CLAUDE.md, PROGRESS.md, and workflow docs to reflect an **MCP-first** architecture. The RAG pipeline is exposed as MCP tools consumed by Claude Code. The web chat UI becomes Module 9.

---

## Context

The project is pivoting from a web-app-first to an MCP-first approach:
- Claude Code is the primary client — no LLM API key needed for reasoning
- Embeddings via OpenRouter (cheap model)
- The same core Python functions power both MCP tools and the Module 9 FastAPI web layer
- Supabase local setup is already completed

---

## Revised Module Structure

### Module 1: MCP Server Scaffold + Supabase Verification
- Verify local Supabase is healthy (already set up)
- Python MCP server skeleton (stdio transport) with a `hello` tool
- Register as a Claude Code MCP server in project settings
- Verify Claude Code can call the tool
- **Learn:** What MCP is, how tools work, stdio transport, server registration

### Module 2: Document Ingestion via MCP
- `ingest_file` MCP tool — accepts file path, reads content, chunks text
- Chunking strategy (recursive character splitting)
- Embed chunks via OpenRouter (cheap model)
- Store in pgvector
- Pydantic schemas for Document and Chunk
- Supabase Storage for raw files
- `list_documents` and `delete_document` tools
- **Learn:** Chunking, embeddings, vector storage, MCP tool design

### Module 3: Retrieval Tool
- `search_documents` MCP tool — vector similarity search
- Relevance threshold filtering
- Return ranked chunks with source attribution
- **Learn:** Vector search, cosine similarity, relevance scoring

### Module 4: Record Manager
- SHA-256 content hashing on ingest
- Deduplication: skip if hash unchanged
- Re-embed on file modification, delete orphan chunks
- **Learn:** Why naive ingestion duplicates, incremental updates

### Module 5: Metadata Extraction
- LLM structured extraction (via OpenRouter) from first N chars
- `DocumentMetadata` Pydantic schema
- Metadata-filtered retrieval in `search_documents`
- **Learn:** Structured extraction, schema design, metadata-enhanced retrieval

### Module 6: Multi-Format Support
- Docling integration for PDF, DOCX, HTML, Markdown
- Cascade deletes (delete document → delete chunks)
- **Learn:** Document parsing challenges, format handling

### Module 7: Hybrid Search + Reranking
- BM25 keyword search
- Hybrid: vector + keyword, RRF fusion
- Reranker (local bge-reranker or Cohere)
- **Learn:** Why vector alone isn't enough, hybrid strategies

### Module 8: Additional Tools
- `query_sql` MCP tool — text-to-SQL with read-only Postgres user
- `web_search` MCP tool — fallback when docs don't have the answer
- Claude Code handles tool routing natively
- **Learn:** Multi-tool agents, structured vs unstructured data

### Module 9: Web Chat UI
- React + Vite + Tailwind + shadcn/ui frontend
- FastAPI backend wrapping the same core functions MCP tools use
- Supabase Auth (login/signup, RLS)
- Chat UI with threads, SSE streaming
- Ingestion UI with drag-drop upload + Supabase Realtime status
- **Learn:** Building a web interface on top of an existing RAG backend, auth, streaming

---

## Key Architecture Changes

| Aspect | Original | MCP-First |
|--------|----------|-----------|
| Primary interface | React web app | Claude Code via MCP |
| LLM for reasoning | OpenAI/Gemini/OpenRouter | Claude (via Claude Code) — no key needed |
| Embeddings | OpenAI text-embedding-3-small | OpenRouter (cheap model) |
| Chat history | Managed by backend | Managed by Claude Code (M1-8), by FastAPI (M9) |
| Tool routing | Custom agent router | Claude Code selects tools natively |
| Sub-agents | Custom sub-agent spawning | Dropped — Claude Code handles orchestration |
| Web UI | Module 1 (primary) | Module 9 (additional feature) |
| Auth + RLS | From Module 1 | Module 9 only (MCP is single-user local) |
| SSE streaming | From Module 1 | Module 9 only |
