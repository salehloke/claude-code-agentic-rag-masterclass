# Module 2: Document Ingestion via MCP

⚠️ **Medium** — Multiple moving pieces (chunking, embeddings API, pgvector storage, file storage, 3 MCP tools) but each step is well-defined.

## Objective

Build MCP tools for document ingestion, listing, and deletion. Files are chunked, embedded via OpenRouter, and stored in pgvector. Raw files go to Supabase Storage.

---

## Prerequisites

- [ ] Module 1 complete (MCP server running, Supabase connected)
- [ ] OpenRouter API key in `.env` (`OPENROUTER_API_KEY`)

---

## Tasks

### Task 1: Database Schema (Migrations)

- [ ] Create migration: `supabase migration new create_documents_table`
  ```sql
  create table documents (
    id uuid primary key default gen_random_uuid(),
    user_id uuid default auth.uid(),
    filename text not null,
    file_path text,
    content_hash text,
    chunk_count int default 0,
    status text default 'pending' check (status in ('pending', 'processing', 'completed', 'failed')),
    created_at timestamptz default now(),
    updated_at timestamptz default now()
  );

  alter table documents enable row level security;
  create policy "users see own documents" on documents
    for all using (auth.uid() = user_id);
  ```

- [ ] Create migration: `supabase migration new create_chunks_table`
  ```sql
  create table chunks (
    id uuid primary key default gen_random_uuid(),
    document_id uuid references documents(id) on delete cascade not null,
    user_id uuid default auth.uid(),
    content text not null,
    embedding vector(768),
    chunk_index int not null,
    token_count int,
    created_at timestamptz default now()
  );

  alter table chunks enable row level security;
  create policy "users see own chunks" on chunks
    for all using (auth.uid() = user_id);

  -- Vector similarity search index
  create index chunks_embedding_idx on chunks
    using ivfflat (embedding vector_cosine_ops)
    with (lists = 100);
  ```

- [ ] Apply migrations: `supabase db reset`

**Validation**: Tables exist in Supabase Studio, RLS is enabled, vector index created.

### Task 5: `ingest_file` MCP Tool

- [ ] Add to `server/main.py`:
  ```python
  @mcp.tool()
  def ingest_file(file_path: str) -> dict:
      """Ingest a file: read → chunk → embed → store in pgvector."""
  ```
- [ ] Implementation flow:
  1. Read file content from `file_path`
  2. Upload raw file to Supabase Storage
  3. Create `documents` record (status: processing)
  4. Chunk the content using `recursive_split()`
  5. Batch embed chunks via OpenRouter
  6. Insert chunks with embeddings into `chunks` table
  7. Update document status to `completed`, set `chunk_count`
  8. Return summary: `{ document_id, filename, chunk_count, status }`
- [ ] Handle errors: set document status to `failed` on exception

**Validation**: Call `ingest_file` from Claude Code with a test `.txt` file → document and chunks appear in Supabase Studio.
